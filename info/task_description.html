If you want to play a golf on book pages, then as first you need to find holes. Let's do it.

You are given a text as a sequence of strings. If we will place them in column in the order how they are given up to down, then we get an 2D array, but with various length rows. A hole is a single whitespace which are surrounded by letters or digits or punctuation symbols - up, down, left, right and both diagonals. Two or more spaces are not a hole. All "outer" characters consider as whitespaces, so a space on the can not be a hole. Your goal is count how many holes in the given text.

Let's look an example: 

How are you doing?
I'm fine. OK.
Lorem Ipsum?
Of course!!!
1234567890
0        0
1234567890
Fine! good buy!

Here we can find 3 (three) holes: before "OK"; between "Lorem" and "Ipsum"; inside "Of course!!!".

In this mission the main goal to make your code as short as possible. The shorter your code, the more points you earn. Your score for this mission is dynamic and directly related to the length of your code. For reference, scoring is based on the number of characters used. 350 characters is the maximum allowable and it will earn you zero points. For each character less than 350, you earn 1 point. For example for 300 character long code earns you 50 points.

Input: A text as a list of strings (unicode for py2).

Output: The number of single holes as an integer.

Example: 

golf(["How are you doing?",
      "I'm fine. OK.",
      "Lorem Ipsum?",
      "Of course!!!",
      "1234567890",
      "0        0",
      "1234567890",
      "Fine! good buy!"]) == 3

How it is used: Let's check your creativity and python tricks in works with various data structures.

Precondition:
0 < len(text) ≤ 20
all(0 < len(row) ≤ 20 for row in text)
